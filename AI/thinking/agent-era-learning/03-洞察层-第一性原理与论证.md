# 学习笔记：洞察层（第一性原理、批判性思维、论证分析）

文献锚点：SEP 中亚里士多德「本原」与认识次序；Paul–Elder 批判性思维框架；Toulmin (1958)。下面是**怎么用**这些工具拆模型输出、定需求，不是哲学导论。

## 30 秒心智模型

**洞察层解决的是：题是不是好题、结论凭什么站住、你有没有在替噪声买单。**  
自动化把「生成答案」变便宜以后，贵的是**问题陈述**和**论证质量**。第一性帮你拆硬约束与想当然；批判性思维给你检查清单；Toulmin 给的是**非形式论证的骨架**——模型最擅长的恰恰是「听起来像论证的叙述」。

## 这层在解决什么

模型擅长给**答案**，不承担「这题值不值得做」。这一层练两件事：

1. **问题侧**：症状、根因候选、可验证的「若解决 X，凭什么说好」——Agent 能列候选，**拍板优先级**仍是人。  
2. **论证侧**：结论、依据、隐藏假设、限定条件——模型能模仿学术腔，**不等于**证据链完整。

文献分工：

- **第一性 / 本原**：亚里士多德区分**对我们更清楚**与**本身更清楚**， inquiry 从前者走向后者。[1] 工程里「第一性」常被滥用成「无视经验」。经验常常是**便宜的外部约束**；该动刀的是**没说出口的假设**和**被当成物理定律的惯例**。  
- **批判性思维（Paul–Elder）**：推理拆成目的、假设、观点、证据、概念、推论、意涵，并用清晰度、相关性等标准审。[2]  
- **Toulmin**：数据、主张、依据、限定、反驳、支持——对付真实世界里的论证，而不是三段论习题。[3]

## 核心概念：再往下一层

### 第一性：拆的是「假设」，不是「同事」

实用问法：这件事里，哪些是**硬约束**（法律、算力、预算、组织承诺），哪些是**偏好**，哪些是**历史偶然**？  
硬约束错了，后面全是空谈；把偏好当硬约束，会抬杠。哲学里对 *archê* 的讨论提醒：**本原**在解释里要同时承担逻辑与说明角色，不是随便贴标签。[1]

### Toulmin 骨架：先找「主张」，再找「依据」

模型长回答里，**看起来像结论的句子**往往是主张；**举例、数据、引用**是数据；**从数据到主张的桥**经常没写清——那就是依据薄弱。[3]  
限定（qualifier）和反驳（rebuttal）在模型输出里常被省略：「在什么条件下结论不成立？」这一句不问，就容易被流畅度带跑。

### Paul–Elder：当检查表用，别当口号

八要素里，**假设**和**意涵**最容易漏：模型很少主动列出「若这条假设错，整段垮掉」。[2]  
「批判性」不是「多怼几句」，而是**对推理质量下判断**——哪一步弱，弱在哪类标准（清晰度？相关性？逻辑？）。

### 「发现问题」≠「值得做的问题」

可操作的陈述要长这样：**若做 X，在指标 Y、对人群 Z、在时间 T 内可观察地变好**。  
文献不替你选项目，但论证结构会逼你把**症状 / 根因假设 / 可验证预测**分开。[2][3]

## 微型案例：用 Toulmin 拆一句模型输出

**模型说**：「建议采用微服务架构，因为可扩展。」

| Toulmin 项 | 内容 |
|------------|------|
| 主张 | 应采用微服务 |
| 数据 | （往往缺失：当前负载、团队规模、发布频率） |
| 依据 | 「可扩展」——这是** warrant**，需要支撑：扩展的是哪类瓶颈？ |
| 限定 | 在什么规模下成立？小到三人团队可能纯增加运维面 |
| 反驳 | 若组织连单体都测不稳，拆服务会不会放大故障面？ |

拆完你会问出一堆**真实信息需求**，而不是继续让模型扩写「微服务最佳实践」。

## 和 Agent 协作时的落点

| 场景 | 做法 |
|------|------|
| 需求含糊 | 先写「非目标」与「成功判据」，再让模型扩方案 |
| 引经据典 | 要求可核对来源；标出事实 / 推断 / 猜测 |
| 方案争论 | 决策表：选项 × 维度 × 权重，看分歧在信息还是在权重 |
| 输出太多 | 先要问题树或假设列表，再展开细节 |

## 易混辨析

- **怀疑一切 ≠ 批判性思维**。后者有**标准和可重复的拆解**。[2]  
- **第一性 ≠ 抬杠**。它问的是**哪些约束真的硬**，不是否定所有先例。  
- **Toulmin ≠ 写论文格式**。它是**读模型输出时的防呆工具**。[3]

## 最小练习（任选一个）

1. 五句话写清一个需求：**背景、非目标、成功标准、最大风险、第一个验证实验**。  
2. 拿模型一段 200 字回答做 Toulmin 标注：主张、数据、依据、限定至少各一行。[3]  
3. 写一个你坚信的结论，再写**反方**；列出能推翻你的证据类型。
