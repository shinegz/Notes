# E2E 测试需求输入规范

> 本规范定义了 E2E 测试任务的输入格式要求，确保测试执行高效、结果准确。

---

## 1. 必需要素

| 要素 | 说明 | 示例 |
|------|------|------|
| **测试场景** | 清晰的用户操作流程 | "新客有推荐品创建流程" |
| **前置条件** | 用户状态、数据状态 | "用户未创建过 AM Campaign" |
| **操作步骤** | 详细的点击/输入操作 | "进入创建页 → 点击 Add Product → 选择商品" |
| **预期结果** | 每步操作的预期状态 | "弹窗打开，商品列表显示" |
| **截图需求** | 需要截图的关键步骤 | "创建成功弹窗、错误提示状态" |

---

## 2. 推荐输入格式

```
场景名称: [场景标识]
用户状态: [新客/老客/特定条件]
前置数据: [需要的测试数据]

测试步骤:
1. [操作] → [预期结果]
2. [操作] → [预期结果]
...

关键验证点:
- [关键元素/状态]
- [关键元素/状态]
```

---

## 3. 完整示例

### 示例: 新用户创建流程（通用格式）

```
场景名称: [场景标识] - [场景名称]
用户状态: 新用户（未创建过任何内容，无历史数据）
前置数据: 系统有 [N] 个推荐数据项

测试步骤:
1. 导航到 /#!/[创建页路由] → 页面加载成功，显示新用户引导界面
2. 检查推荐数据区域 → 显示 [N] 个推荐卡片
3. 点击 "[添加按钮文案]" → 打开选择弹窗
4. 选择 [M] 个额外数据项 → 已选中状态，计数更新
5. 修改 [设置项A] 为 "[目标值]" → 设置成功，值显示正确
6. 修改 [设置项B] 为 "[目标值]" → 设置成功，值显示正确
7. 点击 "[提交按钮文案]" → 提交成功，重定向到 [目标页面]

关键验证点:
- 推荐数据列表渲染
- 选择弹窗功能（打开/选中/关闭）
- 设置项 A/B 的输入和保存
- 提交后页面跳转
```

---

## 4. 补充信息（可选但推荐）

| 信息 | 说明 | 对测试的帮助 |
|------|------|--------------|
| **API 依赖** | 需要的后端接口 | 配置 Mock 数据 |
| **UI 选择器提示** | 按钮/元素的文本或 class | 加快选择器定位 |
| **已知限制** | 当前环境限制 | 设定合理预期 |
| **截图需求** | 需要截图的关键步骤 | 报告证据收集 |
