# 多模态大模型框架与训练流程

> 让 AI 不只能读文字，还能看图、听声、说话。

## 30 秒心智模型

**多模态模型的核心问题：如何让不同类型的数据"说同一种语言"？**

文本是序列，图像是二维数组，音频是波形。它们的"语法"完全不同。

解决思路：找一个"翻译官"——把图像编码成文本能理解的向量，或者让不同模态都映射到同一个共享空间。

CLIP 用对比学习做到了这一点；GPT-4V 在此基础上把视觉编码器和大语言模型嫁接起来。

## 1. 为什么需要多模态

### 单一模态的局限

纯语言模型（如 GPT-4）在处理某些任务时力不从心：

| 任务 | 纯语言模型 | 多模态模型 |
|------|------------|------------|
| "这张图里有什么？" | ❌ 无法处理 | ✅ 直接看图 |
| "代码哪里有 bug？" | ✅ 能处理文本代码 | ✅ 还能看流程图 |
| "这段录音说了什么？" | ❌ 无法处理 | ✅ 直接听 |
| 图表理解 | 有限 | 强 |

人类获取信息不只靠文字。看图理解、听声音、看视频是常态。AI 也需要这个能力。

### 多模态的价值

```
信息模态分布

人类日常信息获取（估计）

     视觉 ████████████████████ 约 50-60%
     听觉 █████████ 约 20-30%
     文本 ████ 约 10-20%
     其他 █ 约 5%

AI 如果只理解文本，就丢失了大部分信息
```

## 2. CLIP：视觉-语言的桥梁

CLIP（Contrastive Language-Image Pre-training）是多模态学习的里程碑，由 OpenAI 于 2021 年发布。

### 核心思想

传统视觉模型需要人工标注的图像标签。CLIP 改变了这个范式：

> 用自然语言描述来监督视觉学习。

训练数据：互联网上发现的图像-文本对（4 亿对）。

### 对比学习框架

```
CLIP 双塔结构

         图像                    文本
           │                      │
           ▼                      ▼
    ┌─────────────┐        ┌─────────────┐
    │ Image       │        │ Text        │
    │ Encoder     │        │ Encoder     │
    │ (ViT)       │        │ (Transformer)│
    └──────┬──────┘        └──────┬──────┘
           │                      │
           ▼                      ▼
    ┌─────────────┐        ┌─────────────┐
    │ I₁, I₂, ... │        │ T₁, T₂, ... │
    │ (图像向量)    │        │ (文本向量)    │
    └──────┬──────┘        └──────┬──────┘
           │                      │
           └──────────┬───────────┘
                      │
                      ▼
             ┌─────────────────┐
             │   对比学习损失    │
             │                 │
             │   拉近配对的      │
             │   I-T 距离       │
             │                 │
             │   推远不配对的    │
             │   I-T 距离       │
             └─────────────────┘
```

### 损失函数

CLIP 使用对称交叉熵损失，同时优化两个方向：

```
CLIP 损失

L = (1/2) · [L_image_to_text + L_text_to_image]

其中:
L_image_to_text = - Σᵢ log( exp(sim(I_i, T_i)) / Σⱼ exp(sim(I_i, T_j)) )
L_text_to_image = - Σⱼ log( exp(sim(I_j, T_j)) / Σᵢ exp(sim(I_i, T_j)) )

sim(I, T) = cos(Encoder_Image(I), Encoder_Text(T))
            余弦相似度
```

### CLIP 学到了什么

训练完成后，CLIP 建立了图像和文本的语义对齐：

```
CLIP 的能力

图像检索文本:
  输入图像 → 编码 → 与候选文本向量匹配 → 最相似的文本

零样本图像分类:
  输入图像 → 编码
  查询: "a photo of a cat", "a photo of a dog", ...
  返回概率最高的类别

不需要在 ImageNet 上训练，直接迁移！
ImageNet 零样本 top-1 准确率: 76.2%
（对比监督训练 SOTA: 88.4%）
```

### CLIP 的局限

1. **长文本描述处理弱**：CLIP 训练时文本平均只有几个词
2. **空间关系理解有限**："在左边"vs"在右边"容易混淆
3. **计数能力弱**：数图中物体数量经常出错

## 3. 多模态架构：从融合到原生

### 架构分类

```
多模态模型架构谱系

融合方式 1: 嫁接式 (Late Fusion)

    ┌─────────┐        ┌─────────┐
    │  Image  │        │  Text   │
    │ Encoder │        │ Encoder │
    └────┬────┘        └────┬────┘
         │                  │
         └────────┬─────────┘
                  ↓
           ┌───────────┐
           │  融合层    │
           │ Projector │
           └─────┬─────┘
                 ↓
           ┌───────────┐
           │    LLM    │
           └───────────┘

代表: LLaVA, BLIP-2, InstructBLIP

─────────────────────────────────────────────────

融合方式 2: 原生多模态 (Early Fusion)

    ┌─────────────────────────────────────────┐
    │           Unified Transformer            │
    │                                          │
    │  Image → Patch Embed → Transformer      │
    │  Text  → Token Embed → Transformer      │
    │                       │                  │
    │                  早期融合                 │
    └─────────────────────────────────────────┘

代表: Flamingo, GPT-4V, Gemini
```

### 嫁接式架构详解

**LLaVA** 是这类架构的代表作：

```
LLaVA 架构

    图像输入
      │
      ↓
    ┌─────────────────┐
    │ CLIP ViT-L/14   │ ← 冻结预训练视觉编码器
    │ (视觉 backbone) │
    └────────┬────────┘
             │
             ↓
    ┌─────────────────┐
    │ Linear Projector │ ← 可训练: 视觉→语言对齐
    └────────┬────────┘
             │
             ↓
    [IMG_TOKENS] + [USER_MSG] + [ANSWER]
             │
             ↓
    ┌─────────────────┐
    │ Vicuna / LLaMA │ ← 冻结或微调语言模型
    │ (语言 backbone) │
    └─────────────────┘
```

### 原生多模态架构

**Flamingo** 采用早期融合策略：

```
Flamingo 架构

输入层 (Perceiver Resampler)

    Image Patches ──┐
    (若干帧)        ├─► Perceiver ──► 固定数量 visual tokens
                    │
    Text Tokens ────┘

         │
         ↓
    ┌────────────────────────────────────────────────────┐
    │            GATED XATTN-DENSE Layers                │
    │                                                     │
    │  每隔一层 transformer 插入 cross-attention 层      │
    │  视觉信息通过 gating 机制注入语言模型               │
    │                                                     │
    │  h_t = h_t + tanh(gate_t) · CrossAttn(h_t, v)     │
    └────────────────────────────────────────────────────┘
                          │
                          ↓
                  冻结的语言模型 + 新增的交叉注意力层
```

```

    ┌────────────────────────────────────────────────┐

    │  ├── 读懂表格数据                               │
    │  └── 识别手写内容                               │
    ├────────────────────────────────────────────────┤
    │  多图像处理                                     │
    │  ├── 对比多张图像                               │
    │  ├── 关联图像间信息                             │

    ├────────────────────────────────────────────────┤
    │  应用场景                                       │

    │  └── 代码截图→代码解释                          │
    └────────────────────────────────────────────────┘
```

### 技术推测

OpenAI 未公开 GPT-4V 的技术细节。根据社区分析和论文，可以推测：

```
GPT-4V 可能的架构

推测方案 A: 视觉编码器 + 嫁接 + GPT-4

推测方案 B: Vision Token + 原生 GPT-4

关键技术点：

1. 视觉编码
   - 可能使用专门的视觉编码器（如 EVA-CLIP）
   - 或复用 GPT-4 的部分能力

2. 视觉 Token 处理
   - 图像 Patch → Visual Token
   - Token 数量需要控制（计算量爆炸问题）

3. 图文融合
   - 跨模态注意力机制
   - 文本和图像 token 序列交错
```

## 5. 多模态训练流程

### LLaVA 的三阶段训练

LLaVA 是开源社区影响力最大的多模态模型之一，采用三阶段训练：

```
LLaVA 训练流程

┌─────────────────────────────────────────────────────────┐
│                                                         │
│  Stage 1: 语言-图像对齐 (LLM 冻结)                      │
│  ┌───────────────────────────────────────────────────┐ │
│  │  目标: 让 Projector 学会把视觉编码映射到 LLM 空间   │ │
│  │  数据: CC3M (59.8万 图-文对)                       │ │
│  │  冻结: CLIP Encoder + LLM                        │ │
│  │  训练: Linear Projector                           │ │
│  └───────────────────────────────────────────────────┘ │
│                        │                                │
│                        ▼                                │
│  Stage 2: 指令微调 (Projector 冻结)                    │
│  ┌───────────────────────────────────────────────────┐ │
│  │  目标: 让模型学会回答各种视觉问答                    │ │
│  │  数据: LLaVA-Instruct (15.8万)                     │ │
│  │  ├── 对话数据: 用户提问 + 图片 + 回答               │ │
│  │  ├── 详细描述: 图片详细描述数据                      │ │
│  │  └── 复杂推理: 需要逻辑推理的视觉问答                │ │
│  │  冻结: CLIP + LLM                                 │ │
│  │  训练: Linear Projector                           │ │
│  └───────────────────────────────────────────────────┘ │
│                        │                                │
│                        ▼                                │
│  Stage 3: 全量微调 (可选，高端硬件)                    │
│  ┌───────────────────────────────────────────────────┐ │
│  │  目标: 进一步提升效果                              │ │
│  │  冻结: CLIP                                       │ │
│  │  训练: Projector + LLM (QLoRA)                    │ │
│  └───────────────────────────────────────────────────┘ │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 训练数据构建

高质量的多模态训练数据是关键：

```
多模态数据来源

1. 图像-标题对
   - LAION-5B (开源)
   - COYO
   - 网页 alt-text

2. 视觉问答 (VQA)
   - VQAv2
   - GQA
   - OK-VQA (需要外部知识)

3. 指令微调数据
   - GPT-4V 生成合成数据
   - 人工标注
   - 数据清洗和过滤

数据清洗的重要性：
- 图像-文本不匹配
- 图像质量低
- 文本含有噪声（HTML标签、特殊字符）
```

### MiniGPT-4 的捷径

MiniGPT-4 发现了一个有趣的现象：CLIP + LLM 的直接嫁接效果不好，但只需一小步微调就能出奇迹：

```
MiniGPT-4 训练策略

第一阶段（对齐）:
- 冻结 CLIP + LLM
- 只训练一个投影层
- 数据: Conceptual Captions (56万)
- 结果: 图像有响应，但输出混乱

第二阶段（精细化）:
- 用 GPT-3.5 过滤/生成高质量对话
- 小规模高质量数据 (3,500 对话)
- 结果: 自然流畅的多模态对话

关键洞察：大量粗数据 → 小量精数据 = 好的效果
```

## 6. 视频与音频多模态

### 视频多模态

视频比图像多了时间维度：

```
视频表示

视频 = 帧序列 × 时间维度

           t=1       t=2       t=3       t=4
        ┌─────┐   ┌─────┐   ┌─────┐   ┌─────┐
        │     │   │     │   │     │   │     │
        │帧1  │ → │帧2  │ → │帧3  │ → │帧4  │ → ...
        │     │   │     │   │     │   │     │
        └─────┘   └─────┘   └─────┘   └─────┘
           │         │         │         │
           ▼         ▼         ▼         ▼
        tokens    tokens    tokens    tokens
           └────────┴────────┴─────────┘
                   │
                   ▼
            Video Tokens (稀疏采样)

稀疏采样策略:
- 均匀采样: 每 N 帧取 1 帧
- 语义采样: 基于内容变化检测关键帧
```
