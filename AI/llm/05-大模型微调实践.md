# 大模型微调实践

> 不是每个人都有千卡集群，但每个人都能微调出自己的专属模型。

## 30 秒心智模型

**微调的本质：用你的数据，在预训练模型的基础上做"定向强化"。**

全参数微调就像重写整本教材；LoRA 像在书页边缘贴便签——改动少但精准有效。

主流的 PEFT（参数高效微调）方法让普通开发者也能在消费级 GPU 上微调几十亿参数的模型。

## 1. 为什么需要微调

### 预训练模型的局限

预训练模型学到了通用能力，但：

- **知识时效性**：训练数据有截止日期，新知识学不到
- **领域适应性**：医学、法律、代码等专业领域需要专门训练
- **风格控制**：输出格式、语气、偏好难以仅靠 Prompt 控制
- **成本**：每次 Query 都走大模型，延迟和成本高

### 微调的典型场景

```
微调 vs Prompt 工程的选择

适合 Prompt 工程的场景:
├── 通用对话和问答
├── 一次性任务
├── 快速原型验证
└── 少量样本可解决问题

适合微调的场景:
├── 需要稳定输出的任务
├── 领域专业知识注入
├── 特定输出格式/风格
├── 低延迟/离线部署需求
└── 私有数据训练
```

### 微调能解决的问题

```
微调效果示例

任务: 医疗病历摘要

Prompt 工程 (GPT-4):
  输入: [病历内容]
  Prompt: "请用专业术语总结以下病历"
  问题: 可能有医疗术语错误、格式不规范

微调后:
  输入: [病历内容]
  输出: 符合医院规范的专业摘要
  优势:
  ├── 术语准确（用领域数据训练）
  ├── 格式统一（按医院模板）
  └── 延迟低（可部署小模型）
```

## 2. 全参数微调 vs PEFT

### 全参数微调的问题

```
全参数微调的资源消耗

以 LLaMA-7B 为例：

模型参数:      7B
梯度 FP16:    14 GB
优化器状态:   28 GB (Adam 需要 2x 参数量)
模型权重:     14 GB
────────────────────────
总计:        ~56 GB

需要: 至少一张 A100 80GB

以 LLaMA-65B 为例：
总计: ~520 GB
需要: 至少 8 张 A100 80GB
```

除了显存，全参数微调还有：

1. **训练时间长**：需要多 epoch 遍历数据
2. **容易遗忘**：大规模更新可能覆盖预训练知识
3. **部署困难**：每个任务一个完整模型

### PEFT 的核心思想

PEFT（Parameter-Efficient Fine-Tuning）的目标是：**只训练一小部分参数，达到接近全参数微调的效果。**

```
PEFT vs 全参数微调

全参数微调:
  预训练模型 ─────────────────────────────────► 微调后模型
  [所有参数参与训练]

PEFT:
  预训练模型
  [冻结大部分参数]
      │
      ├──► 新增少量参数 (Adapter / LoRA)
      │         │
      │         ▼
      │    只训练这部分
      │
      └──► 微调后模型
           [预训练参数保持冻结 + 新增参数]

参数效率:
LLaMA-7B + LoRA (r=8): ~8M 可训练参数
                        vs 7B 全参数
                        = 0.1% 的参数
```

## 3. LoRA：低秩适配器

### 核心原理

LoRA（Low-Rank Adaptation）的灵感来自神经网络的"过剩参数"现象：

> 训练时，权重矩阵的更新往往不需要满秩。有效信息可能只分布在低秩子空间中。

```
LoRA 数学原理

预训练权重矩阵: W₀ ∈ R^(d×k)

全参数更新:     W = W₀ + ΔW
               其中 ΔW ∈ R^(d×k)，需要学习 d×k 个参数

LoRA 分解:      ΔW = B·A
               其中 B ∈ R^(d×r), A ∈ R^(r×k)
               r << min(d, k)

最终:           W = W₀ + B·A
               前向传播: h = W₀x + BAx

只训练 A 和 B，参数从 d×k 降到 (d×r + r×k)
```

### 秩的选择

```
LoRA 秩 (r) 的影响

假设: d=4096, k=4096

r=4:   A(4×4096) + B(4096×4) = 32,768 参数
r=8:   A(8×4096) + B(4096×8) = 65,536 参数
r=16:  A(16×4096) + B(4096×16) = 131,072 参数
r=64:  A(64×4096) + B(4096×64) = 524,288 参数

全参数: 4096 × 4096 = 16,777,216 参数

常见配置:
- r=8:   轻量微调，快速实验
- r=16:  平衡效果和效率
- r=64:  追求更高质量
```

### LoRA 目标层

默认情况下，LoRA 只作用于 Attention 层：

```
LoRA 应用位置

Transformer 层:
  Input
    │
    ▼
  ┌─────────────────────────────────────────────┐
  │  Self-Attention                             ││
  │                                             │
  │    Q ──► Linear ──► Q'                     │
  │           │                                 │
  │           ▼ LoRA (可选)                      │
  │       Q_lora = W₀ + B·A                    │
  │                                             │
  │    K ──► Linear ──► K'                     │
  │    V ──► Linear ──► V'                     │
  │    O ──► Linear ──► O'                     │
  └─────────────────────────────────────────────┘
    │
    ▼
  FFN (通常不加 LoRA)
```

### LoRA 超参数详解

```python
# LoRA 配置示例
lora_config = {
    "r": 16,                    # 秩，越大越强但越慢
    "lora_alpha": 32,           # 缩放因子，通常 = 2×r
    "lora_dropout": 0.05,       # Dropout 防止过拟合
    "target_modules": [         # 应用 LoRA 的层
        "q_proj",
        "v_proj", 
        "k_proj",
        "o_proj",
        # 可选添加
        # "gate_proj",
        # "up_proj", 
        # "down_proj",
    ],
    "bias": "none",             # 不训练 bias
    "task_type": "CAUSAL_LM",   # 因果语言模型
}
```

## 4. QLoRA：量化+LoRA

### 核心思想

QLoRA（Quantized LoRA）的突破在于：**量化模型权重到 4-bit，用 LoRA 高效微调。**

```
QLoRA vs LoRA

LoRA:
  模型权重: FP16 (16-bit)
  可训练: LoRA 参数 (FP16)
  需要显存:  ~56 GB (LLaMA-7B)

QLoRA:
  模型权重: NF4 (4-bit) + 解量化
  可训练: LoRA 参数 (FP16)
  需要显存:  ~6 GB (LLaMA-7B)

显存降低: 56 GB → 6 GB
可以在 RTX 3090 (24GB) 上微调 LLaMA-7B！
```

### NF4 数据类型

NF4（4-bit NormalFloat）是针对神经网络权重分布设计的新型量化格式：

```
量化 vs NF4

标准量化:
  把连续值映射到离散等级
  问题: 均匀切分不匹配权重分布

NF4:
  设计时考虑权重的分位数分布
  4-bit = 16 个等级
  更好地保留重要信息

解量化:
  存储的是 4-bit 索引
  实际计算时反量化回 FP16
  精度损失发生在计算外
```

### QLoRA 内存优化

```
QLoRA 显存占用分析 (LLaMA-7B)

组件                    精度     显存
──────────────────────────────────────
模型权重 (NF4)          4-bit    ~3.5 GB
模型权重 (解量化缓存)    16-bit   ~14 GB
LoRA 参数 (FP16)        16-bit   ~0.1 GB
梯度 (NF4→FP16 反向)    16-bit   ~14 GB
优化器状态 (量化)        8-bit    ~0.6 GB
中间激活值              16-bit   ~6 GB
──────────────────────────────────────
总计                              ~38 GB

实际配置优化后可降到 ~6-10 GB
```

## 5. 其他 PEFT 方法

### Adapter

```
Adapter 架构

原始 Transformer:
  Input → Self-Attn → Add&Norm → FFN → Output

Adapter 版本:
  Input → Self-Attn → Add&Norm → 
                              │
          ┌───────────────────┤
          ▼                   │
    ┌───────────┐              │
    │  DownProj │ ──► ReLU ──► UpProj ──┤
    │  (d→r)    │              │         │
    └───────────┘              │         │
          │                    ▼         │
          └──────────► Add ──► Norm ──► FFN

Adapter 参数: 2×d×r + d×r ≈ 3×d×r
```

### Prefix Tuning

```
Prefix Tuning vs LoRA

LoRA: 在权重矩阵上添加低秩分支
      训练参数: A, B 矩阵

Prefix Tuning: 在输入序列前添加可学习的前缀

    ┌─────────────────────┬─────────────────────┐
    │     P₁ P₂          │    Input Tokens      │
    │  (可学习的前缀)     │                     │
    └─────────────────────┴─────────────────────┘

Prefix 参数: N × d × 2 (两层 MLP)
适合: Prompt 工程风格的任务
```

### QLoRA vs LoRA vs Adapter vs Prefix Tuning

| 方法 | 可训练参数 | 推理延迟 | 效果 | 显存需求 |
|------|------------|----------|------|----------|
| 全参数 | 100% | 无 | 最好 | 最高 |
| LoRA | 0.1-1% | 轻微增加 | 接近全参数 | 中等 |
| QLoRA | 0.1-1% | 轻微增加 | ~LoRA | 最低 |
| Adapter | 1-5% | 中等 | 良好 | 中等 |
| Prefix | < 0.1% | 无 | 良好 | 低 |

pip install llmtuner

### 数据准备

```
微调数据格式 (Alpaca 格式)

[
  {
    "instruction": "把下面的中文翻译成英文",
    "input": "今天天气真好",
    "output": "The weather is really nice today."
  },
  {
    "instruction": "用一句话总结这段文字",
    "input": "人工智能技术正在快速发展...",
    "output": "AI 技术正快速发展，影响各行各业。"
  }
]

关键要点:
- 指令清晰具体
- 输入和输出配对
- 数量: 几千到几万条均可
- 质量 > 数量
```

### LoRA 微调代码

```python
from transformers import AutoModelForCausalLM, AutoTokenizer, TrainingArguments, DataCollatorForLanguageModeling
from peft import LoraConfig, get_peft_model, prepare_model_for_kbit_training
from datasets import load_dataset
import torch

# 1. 加载模型 (4-bit 量化)
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-7b-hf",
    load_in_4bit=True,
    torch_dtype=torch.float16,
    device_map="auto"
)
model = prepare_model_for_kbit_training(model)

# 2. 配置 LoRA
lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj", "k_proj", "o_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)

# 3. 应用 LoRA
model = get_peft_model(model, lora_config)
model.print_trainable_parameters()
# 输出: trainable params: 4194304 || all params: 6738415616 || trainable%: 0.0623

# 4. 准备数据
dataset = load_dataset("json", data_files="your_data.jsonl")
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-2-7b-hf")

def tokenize(example):
    text = f"### 指令:\n{example['instruction']}\n\n### 输入:\n{example['input']}\n\n### 回答:\n{example['output']}"
    return tokenizer(text, truncation=True, max_length=512)

dataset = dataset.map(tokenize, remove_columns=dataset["train"].column_names)

# 5. 训练
training_args = TrainingArguments(
    output_dir="./lora_output",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    gradient_accumulation_steps=4,
    learning_rate=2e-4,
    warmup_ratio=0.03,
    lr_scheduler_type="cosine",
    logging_steps=10,
    save_steps=100,
    fp16=True,
    optim="paged_adamw_8bit"
)

trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=dataset["train"],
    data_collator=DataCollatorForLanguageModeling(tokenizer, mlm=False)
)

trainer.train()
```

### 模型合并与导出

```python
from peft import PeftModel

# 合并 LoRA 权重到基础模型
base_model = AutoModelForCausalLM.from_pretrained("meta-llama/Llama-2-7b-hf")
model = PeftModel.from_pretrained(base_model, "./lora_output")

# 合并并保存
merged_model = model.merge_and_unload()
merged_model.save_pretrained("./merged_model")

# 或只合并权重，不改变模型结构
# model.merge_and_unload()
```

## 7. 常见问题与解决

### 过拟合

```
问题: 模型在训练数据上表现好，泛化差

信号:
- 训练 loss 很低但验证 loss 上升
- 输出几乎等于训练数据的复制

解决方案:
1. 增加 LoRA 的 r 值（更多表达能力）
2. 降低 learning rate
3. 减少训练 epoch
4. 增加 dropout (lora_dropout)
5. 增加数据量或数据增强
```

### 灾难性遗忘

```
问题: 微调后模型忘记了预训练学到的能力

信号:
- 通用对话能力下降
- 简单的数学/逻辑推理出错

解决方案:
1. 使用更小的 learning rate
2. 只微调少数层（如只微调 q_proj, v_proj）
3. 添加预训练数据混合训练
4. 使用 RLHF 而不是 SFT
5. LoRA 的 r 不要太大
```

### 训练不稳定

```
问题: Loss 爆炸、NaN

解决方案:
1. 使用 bf16 代替 fp16
2. 梯度裁剪 (max_grad_norm=1.0)
3. 预热学习率 (warmup_ratio=0.03)
4. 检查数据质量（特殊字符、空值）
5. 使用 paged_adamw_8bit 优化器
```

### 推理部署

```python
# 推理代码
from peft import PeftModel
import torch

model = AutoModelForCausalLM.from_pretrained(
    "base_model",
    torch_dtype=torch.float16,
    device_map="auto"
)
model = PeftModel.from_pretrained(model, "lora_output")

def chat(prompt):
    inputs = tokenizer(prompt, return_tensors="pt").to(model.device)
    outputs = model.generate(**inputs, max_new_tokens=200, temperature=0.7)
    return tokenizer.decode(outputs[0], skip_special_tokens=True)
```

## 总结

```
PEFT 方法全景

参数高效微调

┌───────────────────────────────────────────────────────┐
│  LoRA 系列                                              │
│  ├── LoRA: 低秩矩阵分解                                │
│  ├── QLoRA: 4-bit 量化 + LoRA                         │
│  └── DoRA: 权重分解 + LoRA (更强)                     │
├───────────────────────────────────────────────────────┤
│  Adapter 系列                                           │
│  ├── Adapter: 瓶颈结构 + 残差                         │
│  └── AdapterFusion: 多 Adapter 融合                   │
├───────────────────────────────────────────────────────┤
│  Prefix/Prompt 系列                                    │
│  ├── Prefix Tuning: 可学习前缀                        │
│  ├── Prompt Tuning: 软提示                            │
│  └── P-Tuning: LSTM/MLP 编码器                        │
├───────────────────────────────────────────────────────┤
│  选择建议:                                             │
│  ├── 资源有限: QLoRA (4-bit, r=8-16)                 │
│  ├── 追求质量: LoRA (r=64) + 全量微调最后几层         │
│  └── 推理优先: LoRA + 合并权重                         │
└───────────────────────────────────────────────────────┘
```

微调的本质是"精准外科手术"——用最小的改动，让模型适应你的任务。LoRA 和 QLoRA 把这件事做到了极致：百元级别成本，小时级别时间，就能训出一个专属模型。
