# 搜集资料策略详解

本文档是 SKILL.md 中流程的细化参考，在需要更具体的关键词设计、分类维度或输出模板时可查阅。

## 1. 搜索策略（多轮、多语言、分层）

### 第一轮：概念与定义
- **目的**：弄清主题是什么、谁提出的、核心术语，便于后续精准搜索。
- **英文查询示例**：`<topic> learning resources tutorial`、`what is <topic>`
- **中文查询示例**：`<主题> 是什么 概念`、`<主题> 学习 教程`
- **产出**：1–2 句概念简述（可写入输出文档开头），以及后续搜索可用的关键词（官方名、人物、平台名）。

### 第二轮：按资源类型分层
- **文档/官方**：`<topic> official documentation`、`<平台> <topic> tutorial`（如 GitHub、Microsoft Learn、Google）。
- **视频**：`<topic> YouTube video tutorial`、`<人物> <topic>`（若第一轮出现关键人物）。
- **课程/学院**：`<topic> course`、`<topic>-academy`、`Coursera/Codecademy <topic>`。
- **中文延伸**：`<主题> 工具 教程 中文`、`<主题> 入门 实战`。

每一轮用 **2–4 个并行 WebSearch**，覆盖不同角度，避免一次只搜一个模糊词。

### 第三轮：补全与验证（按需）
- 若某类资源明显偏少，再补 1–2 次针对性搜索。
- 对关键链接可用 `mcp_web_fetch` 或 `web_fetch` 拉取首屏，确认可访问且内容相关（不必全文解析）。

## 2. 分类维度建议

按「形式 + 受众」组织表格，便于读者按需选用：

| 维度     | 示例分类                         |
|----------|----------------------------------|
| 形式     | 官方文档、视频、在线课程、文章、词条 |
| 语言     | 中文 / 英文 / 中英混合           |
| 深度     | 入门、进阶、专题/实战            |
| 来源     | 官方、平台、社区、个人           |

输出时至少包含：**资源名称、形式、简要说明、链接**。可选：时长、难度、语言。

## 3. 输出结构建议

- **标题**：`<主题> 学习资料汇总`（或用户指定的标题）。
- **开头**：1 段概念简述（2–3 句），方便不了解该主题的人快速建立认知。
- **正文**：按「文档 → 视频 → 课程 → 文章」等分节，每节用表格呈现。
- **结尾**：可加「速查/按人群推荐」（如：零基础推荐 X、系统学习推荐 Y、做项目推荐 Z），或与用户已有笔记的配合说明。
