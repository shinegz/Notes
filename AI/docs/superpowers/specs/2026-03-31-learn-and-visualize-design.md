**日期**: 2026-03-31  
**状态**: 已批准，待实现

---

## 1. 概述

### 1.1 核心定位

learn-and-visualize 是一个指导 Agent 像人类一样学习并生成交互式可视化 HTML 学习笔记的 Skill。它不仅仅是总结资料，而是运用人类优秀的学习策略去**理解、消化、重构**知识，最终以**简洁、美观、实用**的 HTML 形式呈现。

### 1.2 设计原则

- **一图胜千言**: 善用图形、动画解释复杂概念
- **自适应**: 根据主题自动选择最适合的组织方式和交互程度
- **去 AI 味**: 最终文字经过 humanizer-zh 处理，读起来像真人写的
- **实用优先**: 简洁、美观但实用

---

## 2. 触发条件

当用户表达以下意图时触发：

- "去学习 xxx"
- "想了解 xxx，并生成学习笔记"
- "研究一下 xxx，做成可视化笔记"
- "帮我把 xxx 整理成交互式笔记"

**注意**: 与 learning-assistant 的触发语有重叠，当用户明确提到"可视化""交互式""HTML""笔记"等关键词时，优先使用本 Skill。

## 3. 学习方法论

Agent 在消化知识时自动应用以下方法：

### 3.1 费曼技巧
- 用简单语言解释概念
- 识别理解盲区并补充研究
- 确保每个概念都能用一句话讲清楚

### 3.2 概念映射
- 构建知识关联网络
- 识别核心概念和从属概念
- 建立概念间的层级关系

### 3.3 类比学习
- 用读者熟悉的领域映射新概念
- 建立一致的隐喻体系
- 避免随机换比喻

### 3.4 多感官学习
- 视觉: 图形、动画、颜色编码
- 文字: 简练、精准、犀利的描述
- 互动: 可操作的演示、探索式元素

### 3.5 问题导向
- 从核心问题出发
- 逆向寻找知识
- 用问题串联内容

## 4. 资料收集策略

无论用户是否提供资料，Agent 都会执行以下流程：

### 4.1 资料收集流程

```
用户输入
    │
    ├─→ 用户提供资料? ──→ 收集并阅读
    │
    └─→ 识别知识缺口
            │
            ├─→ 主动搜索补充
            │
            └─→ 交叉验证信息
```

### 4.2 搜索策略

采用分层并行搜索：

1. **第一层 - 概念定义**: 获取核心术语和基本概念
2. **第二层 - 多角度搜索**: 官方文档、教程、视频、中文资料
3. **第三层 - 缺口填补**: 针对特定薄弱点深入搜索

### 4.3 信息验证

- 优先使用官方文档和权威来源
- 交叉验证多个来源
- 标注不确定的信息

## 5. 输出规范

### 5.1 文件格式

- **格式**: 单文件 HTML
- **样式**: 嵌入式 CSS
- **脚本**: 嵌入式 JavaScript
- **依赖**: 尽可能使用 CDN，避免本地依赖

### 5.2 视觉风格

提供两种风格供用户选择（预览后决定）：

#### 风格 A: 手绘/草图风
- 特点: 温暖亲切，模拟手写笔记
- 适用: 创意类、概念类、入门类主题
- 技术: Rough.js 实现手绘效果

#### 风格 B: 杂志/媒体风
- 特点: 精心设计的网络长文，图文并茂
- 适用: 技术类、深度分析类主题
- 技术: 现代排版，大图，精致细节

### 5.3 可视化技术栈

根据内容类型自动选择：

| 内容类型 | 推荐技术 | 示例 |
|----------|----------|------|
| 流程/时序 | Mermaid | 算法流程、系统交互 |
| 关系网络 | D3.js | 概念关联、知识图谱 |
| 算法演示 | Canvas/SVG 动画 | 排序过程、数据结构变化 |
| 空间/几何 | Three.js | 3D 结构、空间概念 |
| 交互探索 | 自定义交互组件 | 参数调整、状态切换 |

### 5.4 内容组织方式

Agent 根据主题自动选择最适合的组织方式：

| 组织方式 | 适用场景 | 特点 |
|----------|----------|------|
| 线性叙事 | 有明确先后顺序的主题 | 像一本书，从头到尾阅读 |
| 模块化卡片 | 独立概念组成的主题 | 可跳跃阅读，每个卡片完整 |
| 双轨制 | 既有快速需求又有深度需求 | 提供"快速概览"和"深度探索"两条路径 |
| 问题驱动 | 以解决特定问题为目标 | 以关键问题为线索组织 |
| 概念地图 | 复杂的知识体系 | 先展示整体图谱，再逐层深入 |

### 5.5 交互程度

根据知识类型自动选择：

| 交互程度 | 适用场景 | 示例 |
|----------|----------|------|
| 静态展示 | 概念介绍、历史背景 | 精美的网页文章 |
| 轻度交互 | 需要选择性阅读的内容 | 折叠/展开、标签切换、悬停提示 |
| 深度交互 | 算法、原理类内容 | 参数调整实时反馈、步骤控制 |
| 探索式 | 复杂系统、多维度内容 | 可选择学习路径、互动教程 |

### 5.6 文字风格

所有文字必须经过 humanizer-zh 处理：

- 去除 AI 味（避免"此外""至关重要""凸显"等词汇）
- 变化句子长度和结构
- 使用具体细节而非模糊主张
- 适当使用第一人称，表达真实观点
- 允许一些不完美的表达，增加人性

**humanizer-zh 核心规则速查**（详细规则参考 humanizer-zh skill）：

1. **删除填充短语** - 去除开场白和强调性拐杖词
2. **打破公式结构** - 避免二元对比、戏剧性分段、修辞性设置
3. **变化节奏** - 混合句子长度。两项优于三项。段落结尾要多样化
4. **信任读者** - 直接陈述事实，跳过软化、辩解和手把手引导
5. **删除金句** - 如果听起来像可引用的语句，重写它

## 7. 技术实现要点

### 7.1 HTML 模板结构

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>{主题名} - 学习笔记</title>
  <!-- 嵌入式 CSS -->
  <style>...</style>
  <!-- 外部库 CDN -->
  <script src="..."></script>
</head>
<body>
  <!-- 导航/目录 -->
  <nav>...</nav>
  
  <!-- 主要内容区 -->
  <main>
    <!-- 30秒心智模型 -->
    <section id="overview">...</section>
    
    <!-- 核心内容 -->
    <section id="content">...</section>
    
    <!-- 可视化区域 -->
    <section id="visualization">...</section>
    
    <!-- 总结 -->
    <section id="summary">...</section>
  </main>
  
  <!-- 嵌入式 JavaScript -->
  <script>...</script>
</body>
</html>
```

### 7.2 依赖管理

优先使用 CDN：
- Mermaid: jsDelivr CDN
- D3.js: jsDelivr CDN
- Three.js: jsDelivr CDN
- Rough.js: jsDelivr CDN

### 7.3 性能考虑

- 单文件设计，减少 HTTP 请求
- 懒加载非关键资源
- 动画使用 requestAnimationFrame
- 提供"简化版"选项（减少动画）

## 9. 边界与限制

### 9.1 适用范围

- 适合: 技术概念、算法、系统架构、理论知识
- 不适合: 实时数据、需要后端支持的应用、超大规模内容

### 9.2 技术限制

- 单文件大小建议不超过 5MB
- 复杂 3D 场景可能需要简化
- 部分高级交互在移动设备上可能受限

### 9.3 内容限制

- 不生成需要登录的内容
- 不生成涉及敏感信息的内容
- 引用来源需标注，但正文不依赖外部链接
