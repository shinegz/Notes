# Entity 页模板

> Entity 页的标准结构，确保同一 shelf 下所有实体页结构一致。适用于 `entities/` 下所有页面。

---

## Frontmatter

```yaml
---
title: "实体名"
type: entity
tags: [tag1, tag2]
sources:
  - shelf-name/sources/source-name
last_updated: YYYY-MM-DD
---
```

---

## 标准结构

### 1. 一句话定义（必须）

用一句话概括实体的本质，让读者快速建立心智模型。放在所有内容之前。

```
## 一句话定义

[实体身份] + [核心定位] + [最重要的事实]
```

### 2. 关键属性（推荐用表格）

适用于人物、组织、项目等各类实体。属性因类型而异：

**人物**：

| 属性 | 值 |
|------|-----|
| 职位/身份 | … |
| 所属机构 | … |
| 主要领域 | … |
| 重要荣誉 | …（可选） |

**组织**：

| 属性 | 值 |
|------|-----|
| 类型 | 研究机构 / 公司 / 开源组织 / … |
| 核心产品 | … |
| 重要里程碑 | … |
| 社区资源 | …（可选） |

**项目**：

| 属性 | 值 |
|------|-----|
| 作者/维护者 | … |
| 仓库地址 | … |
| 许可证 | … |
| 核心功能 | … |

### 3. 关键贡献 / 关键产品（可选）

实体最重要的贡献或产品。用列表或表格呈现。

```
## 关键贡献

- **贡献/产品名称** — 说明
```

### 4. Key Quotes（可选）

原文中特别重要的引文。

```
## Key Quotes

> "引文原文" — 上下文说明
```

### 5. 来源

列出本文的参考来源，必须有 WikiLink。

```
## 来源

- [[shelf/sources/source-name|Source Name]] — 参考依据
```

---

## 不同实体类型的 section 选用指南

| 实体类型 | 必须有的 section | 可选的 section |
|----------|-----------------|----------------|
| 人物 | 一句话定义、关键属性、来源 | Key Quotes、主要著作、关键贡献 |
| 组织 | 一句话定义、关键属性、关键产品、来源 | Key Quotes、源码架构、社区资源 |
| 项目/框架 | 一句话定义、关键属性、关键设计决策、来源 | 源码架构、使用示例、关键产品 |
| 学术论文 | 一句话定义、关键属性（发表信息）、主要结论、来源 | Key Quotes、方法论、关键贡献 |

---

## 写作原则

1. **以一句话定义开头**：不能是"XX 是一个..."的平淡介绍，要有一句话的核心事实
2. **属性用表格**：易于对比和查阅；不要用纯文本罗列属性
3. **来源必须带 WikiLink**：`[[...]]` 不能缺少显示文本
4. **区分事实和推断**：entity 页记录事实，不做过度推断；推测性内容加"据推测"等限定词

---

## 检查清单

写完 entity 页后自检：

- [ ] 第一段是"一句话定义"，不是背景铺垫或自我介绍
- [ ] 关键属性用表格呈现
- [ ] `## 来源` 下的 source 都有 WikiLink 且格式正确
- [ ] frontmatter 有 `title`、`type`、`sources`、`last_updated`
