# Concept 页模板

> Concept 页的标准结构，确保同一 shelf 下所有概念页结构一致。适用于 `concepts/` 下所有页面。

---

## Frontmatter

```yaml
---
title: "概念名"
type: concept
tags: [tag1, tag2]
sources:
  - shelf-name/sources/source-name
last_updated: YYYY-MM-DD
---
```

---

## 标准结构

### 1. 一句话理解（必须）

用一句话概括概念的核心，让读者快速建立心智模型。放在所有内容之前。

```
## 一句话理解

[核心定义] + [为什么重要]
```

### 2. 背景与问题（可选）

这个概念解决什么问题？或它的 context 是什么。

```
## 背景

[为什么需要这个概念 / 它解决什么问题]
```

### 3. 核心机制 / 原理

详细解释概念的工作原理。用公式、代码、表格辅助说明。

```
## 核心机制

### 子标题（根据需要添加层级）

[公式]
[代码示例]
[表格]
```

### 4. 类型 / 变体 / 应用（可选）

概念的常见类型、变体或实际应用场景。

```
## 类型

| 类型 | 描述 | 应用 |
|------|------|------|
| A | ... | ... |
| B | ... | ... |
```

### 5. 与其他概念的关系（可选）

这个概念与其他概念的关联。用 WikiLink 引用，不要重复解释。

```
## 关联概念

- [[concepts/other-concept]] — 关联说明
- [[concepts/another-concept]] — 关联说明
```

### 6. 来源

列出本文的参考来源，必须有 WikiLink。

```
## 来源

- [[shelf/sources/source-name|Source Name]] — 参考依据
```

---

## 写作原则

1. **以 Epitome 开头**：第一段必须是一句话概括，不能是"背景介绍"
2. **原子化**：一个 concept 页只解释一个概念；如果需要解释另一个概念，用 WikiLink 引用
3. **用 WikiLink 而不是重复内容**：如果另一个概念页有详细解释，不要在当前页重复，用 `[[...]]` 引用
4. **表格用于对比，不是罗列**：表格用来对比类型/变体，不是用来罗列名词列表
5. **来源必须带 WikiLink**：`[[...]]` 不能缺少显示文本

---

## 常见 section 名称规范

| 内容 | 推荐 section |
|------|-------------|
| 一句话概括 | `## 一句话理解` |
| 解决的问题 | `## 背景` 或 `## 问题` |
| 核心原理 | `## 核心机制` |
| 公式/推导 | `## 核心机制` 下的 `### 公式` |
| 类型/变体 | `## 类型` 或 `## 变体` |
| 应用场景 | `## 应用` |
| 关联概念 | `## 关联概念` |
| 参考来源 | `## 来源` |

**禁止作为 section 标题**：
- 孤立的变量名（如 `# d_k = 64 时`）— 这是内容碎片，不是标题
- 中文问答句（如 `# 人工标注的问答对`）— 这不是标题，是内容残片

---

## 检查清单

写完 concept 页后自检：

- [ ] 第一段是"一句话理解"，不是背景铺垫
- [ ] 所有 `## 来源` 下的 source 都有 WikiLink 且格式正确
- [ ] 没有孤立的标题（如变量名、中文问答句）
- [ ] 引用其他 concept 时用 `[[path|name]]`，不是重复内容
- [ ] frontmatter 有 `title`、`type`、`sources`、`last_updated`
