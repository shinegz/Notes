# Purpose

## 学习目标

- 将日常学习资料编译为**持久、互链的 Markdown wiki**（Karpathy llm-wiki 思路），而非仅问答式 RAG。
- 按**门类（书架）**组织：`raw/<门类>/…` 与 `wiki/<门类>/…` 路径一致。

## 核心问题（由你补充）

- 当前最想深入的一条主线是？
  - **规范驱动开发（Spec-Driven Development）**：AI 时代的软件工程方法论，从理论基础到框架源码实现（Superpowers、Gstack、Compound Engineering 等）

## In-scope

- 可追溯来源的笔记、文章、官方文档、论文文本、高质量博客。
- 基于本地证据撰写概念页、实体页、综合回答。

## Out-of-scope

- 批量突破付费墙、违反站点 ToS 的抓取。
- 镜像超大二进制（视频、大模型权重等）— 使用 `raw/<门类>/refs/` 指针说明即可。
