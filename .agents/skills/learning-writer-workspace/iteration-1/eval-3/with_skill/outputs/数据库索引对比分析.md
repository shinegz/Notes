# 数据库索引对比分析

> 选对索引，让查询快 100 倍；选错索引，让数据库白占空间。

---

## 30 秒心智模型

| 索引类型 | 类比 | 最适合 |
|---------|------|-------|
| **B-Tree** | 通用工具箱 | 范围查询、排序 |
| **Hash** | 精准雷达 | 等值查找 |
| **Bitmap** | 标签系统 | 低基数列、布尔组合 |
| **GiST** | 空间地图 | 地理位置、全文搜索 |

---

## 场景选择指南

```
需要什么？
    │
    ├─ 等值查询 ──────────────────→ Hash 索引
    │
    ├─ 范围查询/排序 ─────────────→ B-Tree 索引
    │
    ├─ 地理位置/几何数据 ──────────→ GiST 索引
    │
    └─ 低基数列 + 布尔组合 ────────→ Bitmap 索引
```

---

## B-Tree 索引

### 核心特点

- **平衡树**：所有叶子节点在同一深度
- **多路分支**：每个节点可有多个子节点
- **有序存储**：数据按 key 排序

### 适用场景

✅ 范围查询：`WHERE age BETWEEN 20 AND 40`
✅ 排序操作：`ORDER BY created_at`
✅ 前缀匹配：`WHERE name LIKE 'A%'`
✅ 等值查询：`WHERE id = 123`

### 性能数据

```
查询性能：O(log n)，约 3-4 次 I/O 就能查完 100 万行
插入性能：O(log n)，自动维护平衡
空间占用：适中，约 1.2-1.5x 数据大小
```

---

## Hash 索引

### 核心特点

- **直接寻址**：通过哈希函数计算位置
- **无序存储**：不保持任何顺序
- **O(1) 查询**：理论上最快

### 适用场景

✅ 等值查询：`WHERE email = 'user@example.com'`
✅ 缓存查找：`WHERE session_id = ?`
✅ 唯一性检查

### 不适合场景

❌ 范围查询：`WHERE score > 60` → 退化为全表扫描
❌ 排序操作：无法提供有序结果
❌ 前缀匹配

---

## Bitmap 索引

### 核心特点

- **位图存储**：每个值用 1 bit 表示
- **极省空间**：低基数列效果最好
- **布尔运算快**：AND/OR 操作极快

### 工作原理

```
用户表（性别列）：

行ID:  1  2  3  4  5  6  7  8
男:    1  0  1  0  1  0  1  0
女:    0  1  0  1  0  1  0  1

查询"男性"：直接返回位图 [1,0,1,0,1,0,1,0]
```

### 适用场景

✅ 低基数列：性别、状态、类型
✅ 布尔组合查询：男性 AND 活跃用户
✅ 数据仓库：OLAP 聚合分析

---

## GiST 索引

### 核心特点

- **通用框架**：支持多种数据类型
- **可扩展**：用户可定义自己的搜索谓词
- **复杂查询**：地理位置、范围、全文

### 适用场景

✅ 地理位置：`WHERE location <-> point '(0,0)' < 100`
✅ 范围查询：`WHERE daterange @> '2024-01-01'`
✅ 全文搜索：`WHERE tsvector @@ query`

### PostgreSQL 示例

```sql
-- 地理距离查询
CREATE INDEX idx_location ON places USING GIST(location);
SELECT * FROM places WHERE location <-> point '(0,0)' < 1000;

-- 全文搜索
CREATE INDEX idx_content ON articles USING GIST(to_tsvector('english', content));
```

## 实战建议

### 电商系统

```sql
-- 用户表
CREATE INDEX idx_user_email ON users USING HASH(email);     -- 登录
CREATE INDEX idx_user_created ON users(created_at);         -- 注册时间查询
CREATE INDEX idx_user_status ON users(status);               -- 状态筛选

-- 订单表
CREATE INDEX idx_order_user ON orders(user_id);             -- 查用户订单
CREATE INDEX idx_order_created ON orders(created_at);       -- 时间范围
```

### 数据仓库

```sql
-- 大表用 BRIN（比 GiST 更省空间）
CREATE INDEX idx_logs_brin ON logs USING BRIN(created_at);

-- 维度列用 Bitmap
CREATE INDEX idx_logs_level ON logs USING BITMAP(level);
```

---

## 总结

1. **通用场景** → B-Tree（最安全）
2. **极速等值** → Hash
3. **低基数布尔** → Bitmap
4. **地理/全文** → GiST
5. **超大数据** → BRIN

没有银弹，根据查询模式选择合适的索引类型。
