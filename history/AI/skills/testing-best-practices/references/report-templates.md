## 测试计划输出模板

Phase 1 完成后输出测试计划，供用户确认。

```markdown
## 测试计划：[功能名]

### 环境
- 技术栈 / 测试框架 / 测试类型

### 测试对象摘要
[一句话说清测什么、为什么测]

### 第一层：单元测试用例
| # | 用例 | 优先级 | 归属模块 |
|---|------|--------|----------|
| 1 | [验证了什么] | 高 | [模块名] |

### 第二层：集成测试用例
| # | 用例 | 协同模块 | 优先级 |
|---|------|---------|--------|
| 1 | [验证了什么] | ModuleA + ModuleB | 高 |

### 第三层：E2E 用例
| # | 场景 | 关键截图点 | 优先级 |
|---|------|-----------|--------|
| 1 | [完整用户流程] | ① 步骤截图 | 高 |

### Mock 与数据策略
- [需 mock 的依赖 + 原因]

### 备注
- [假设、待确认问题]
```

## E2E Mock 数据清单模板

E2E 执行前必须输出 Mock 数据清单，禁止在没有此清单的情况下开始执行。

```markdown
## E2E Mock 数据清单

### 场景 E1: [场景名称]
- **用户状态**：[新客/老客/有数据/无数据]
- **API Mock**：
  - `GET /api/campaigns` → 返回 `[...]`
  - `POST /api/products` → 返回 `{...}`
- **localStorage 注入**：
  - `localStorage.setItem('visited', 'true')`
- **权限开关**：
  - `featureFlag.newCampaign: true`

### 场景 E2: ...
```

### Mock 数据表格形式

| 场景 | API 返回值 | localStorage 状态 | 权限/开关 |
|------|-----------|-------------------|-----------|
| 场景1 | `GET /api/xxx → {...}` | `key: value` | `flag: true` |

## Phase 完成确认模板

### Phase 2A 完成确认

```markdown
## ✅ Phase 2A 完成

- 单元测试文件：[列出所有已创建的测试文件路径]
- 文件组织检查：[确认每个文件对应一个功能模块]
- 执行结果：[通过/失败，失败则说明修复情况]
- 测试报告：[报告文件路径]
```

### Phase 2B 完成确认

```markdown
## ✅ Phase 2B 完成

- 集成测试文件：[列出所有已创建的测试文件路径]
- 协作模块覆盖：[列出被验证的模块协作关系]
- 执行结果：[通过/失败，失败则说明修复情况]
- 测试报告：[报告文件路径]
```

### Phase 2B 跳过确认

如果确认无集成测试，必须输出理由：

```markdown
## ⏭️ Phase 2B 跳过

- 跳过理由：[具体说明为什么各模块不需要协作测试]
- 验证检查点：[列出已检查的信号，确认都不存在]
```

### Phase 2C 完成确认

```markdown
## ✅ Phase 2C 完成

- Mock 数据清单：[确认已输出]
- E2E 脚本文件：[路径]
- 截图目录：[路径，确认截图存在]
- 测试报告：[路径]
- 结果汇总：PASS X / FAIL X / BLOCKED X
```

## 报告命名规范

| 报告类型 | 命名格式 | 示例 |
|----------|----------|------|
| 单元测试报告 | `<feature-slug>-unit-<YYYYMMDD>.md` | `campaign-filter-unit-20260325.md` |
| 集成测试报告 | `<feature-slug>-integration-<YYYYMMDD>.md` | `campaign-filter-integration-20260325.md` |
| E2E 测试报告 | `<feature-slug>-e2e-<YYYYMMDD>.md` | `campaign-filter-e2e-20260325.md` |
