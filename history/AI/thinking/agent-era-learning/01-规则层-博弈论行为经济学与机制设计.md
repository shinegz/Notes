# 学习笔记：规则层（博弈论、行为经济学、机制设计）

定向检索过 Myerson 教材、Kahneman–Tversky (1979)、诺贝尔经济学奖对机制设计的说明；下面是把文献**接到 Agent 规则**上的写法，例子是虚构情境。

## 30 秒心智模型

**规则层解决的是：多方各自想优化自己的东西时，你的条文还能不能让系统别散架。**  
模型、同事、业务方、未来的你，都是能动的主体。博弈论和机制设计给的是语言和倒推；行为经济学提醒你：**写在纸上的理性，落地时往往不像你以为的那样动。**

## 这层在解决什么

能动起来之后，**劲往哪使**由规则决定。这一层要练的是：把含糊愿望写成**别人在局部最优时，全局仍过得去**的约束与激励，而不是只写「要负责任」。

文献里三件事常一起出现：

- **博弈论**：参与人、策略、收益、均衡。Myerson 从决策论收到策略式与贝叶斯博弈，一条反复出现的直觉是：**你没写进收益的东西，执行端会替你猜**——而且经常猜错。[1]
- **行为经济学**：人不是期望效用机。前景理论把价值放在**增益/损失**上，而不是最终财富；概率会被**决策权重**扭曲。[2] 规则若假装人人冷静算账，落地会歪。
- **机制设计**：想要的结果，能不能用一组规则（机制）在均衡里实现？激励相容在诺奖说明里被反复强调。[3] 权限、审批、工具策略、谁报什么类型，都是机制，不是「道德呼吁」。

## 核心概念：再往下一层

### 策略式与均衡：2×2 不是作业，是防呆

至少会画一个 **2×2**：两参与人、各两策略、四个收益格。改一条规则之前问：**纳什均衡动不动？**  
若不动，多半说明条文改的是口号，没改到任何人真正在优化的东西。[1]

容易混的一点：**帕累托更好**和**均衡会过去**，不是一回事。你可以写一条「大家都好」的政策，但若没人有动力遵守，均衡仍在旧点。

### 激励相容（人话版）

机制设计里理想的性质之一是：说真话、按规范做，在均衡里**不亏**。[3] 落到日常：  
「必须人工审核」若意味着审核员背锅、发版的人不背锅，审核就会形式化。规则要问：**在现有激励下，谁会诚实执行？谁会选择糊弄？**

### 前景理论：别只盯 KPI 的「数字变大」

损失比增益更陡：惩罚「失败」往往比奖励「成功」更强烈地扭行为。[2]  
单一 KPI（例如只盯吞吐）会把人推到价值函数的拐角：短期刷满指标，长期风险堆在别处。至少要能说清：**多目标冲突时谁让路、谁升级给人**。

### 多属性取舍：权重不是谦虚，是责任

规则很少只有一个目标。Myerson 开篇就接决策论：在约束下选方案。[1] 实务上写清三件事：**有哪些维度、权重谁拍板、冲突时走哪条升级路径**。没有第三条，扯皮会回到暗处。

## 微型案例：「自动代码审查」政策

设两个参与人：**开发**（想快合并）、**组织**（想少事故）。各两策：开发「严格自测 / 先推再说」；组织「抽查 / 全量卡点」。收益格子不必真算准，**结构**才重要：若组织嘴上要质量、考核只看合并速度，均衡往往滑向「先推再说 + 形式卡点」——这和模型聪明与否无关，和**收益表里写什么**有关。

改规则时问：动的是**考核**、**工具权限**，还是**一句口号**？只有前两者会动均衡。

## 和 Agent 协作时的落点

| 你在做的事 | 规则层要问的 |
|-------------|----------------|
| 写系统提示 / 政策 | 成功长什么样？失败必须怎样暴露？哪些动作永远禁止？ |
| 配工具权限 | 最小权限外，谁在什么条件下能扩权？扩权留不留痕？ |
| 定团队规范 | 奖励什么、惩罚什么？谁会钻空子？钻了谁承担后果？ |

「价值观」要落到**可检查、可推演**的条款，否则模型只能猜你的底线。

## 易混辨析

- **博弈论 ≠ 算计同事**。它给的是**结构化想象**：谁在优化什么。  
- **行为经济学 ≠ 鄙视理性**。它描述的是**可预测偏离**，用来检查条文会不会放大偏误。  
- **机制设计 ≠ 只适用于拍卖**。任何「报类型—分配—付钱/权限」的结构都是机制。

## 最小练习（任选一个）

1. 给常用 Agent 任务加三段：**目标、禁止项、越界时的输出格式**，用一周看误触发是否下降。  
2. 画一个与你工作相关的 2×2，标出你**认为**的均衡；再改一条真实政策，看均衡会不会动。[1]  
3. 找一条「无法执行」的规范，写出**谁在什么情况下会违反、违反的收益是什么**。
